# Introduction to EDR

## What I learned

- Antivirus and EDR are different: traditional antivirus focuses on known threats, while EDR continuously monitors system behavior to detect suspicious activity.
- EDR provides three core capabilities: visibility, detection, and response.
- EDR operates at the host (endpoint) level using agents that collect data and send it to a central console.
- EDR works alongside other security tools as part of a layered defense strategy.
- Detection methods include behavioral analysis, anomaly detection, IOC matching, MITRE ATT&CK mapping, and machine learning.
- EDR supports both automated and manual response actions.

## Key concepts

- Endpoint Detection and Response (EDR): A security solution that monitors, detects, and responds to threats on endpoints.
- EDR Agent: Software installed on endpoints to collect and send data.
- EDR Console: Central platform where endpoint data is analyzed and alerts are managed.

## Why it matters in SOC

- EDR provides deep visibility into endpoint activity and enables faster detection and response to threats.

## Tools / activities used

- Practiced using an EDR console and triaged alerts.

## Personal notes

- The combination of EDR agents and a centralized console makes it easier to detect suspicious behavior and respond quickly to potential threats.
