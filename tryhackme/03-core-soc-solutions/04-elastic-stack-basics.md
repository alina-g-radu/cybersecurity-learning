# Elastic Stack Basics

## What I learned

- The Elastic Stack (ELK) was not originally designed as a SIEM, but many SOC teams use it for log analysis and security monitoring.
- Kibana Query Language (KQL) uses logical operators to filter and search data.

## Key concepts

- Elastic Stack (ELK): A collection of tools used for log collection, analysis, and visualization.
- Elasticsearch: Stores, searches, and analyzes data.
- Logstash: Processes and transforms incoming data.
- Beats: Lightweight agents installed on hosts to collect and send data.
- Kibana: Web-based interface used for visualization, dashboards, and data exploration.
- Index pattern: Defines how data is organized and accessed in Elasticsearch.
- Kibana Query Language (KQL): Query language used in Kibana to search and filter data.

## Why it matters in SOC

- Elastic Stack provides visibility into logs and allows analysts to search, visualize, and investigate security events.

## Tools / activities used

- Practiced using Kibana to analyze logs, create visualizations, and build dashboards.

## Personal notes

- Kibana makes it easier to explore and understand large amounts of log data through visualizations and structured queries.
